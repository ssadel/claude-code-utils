The user wants to see your implementation plan BEFORE you do anything. DO NOT take any actions. No editing files, no writing files, no committing, no staging, no creating anything.

1. Read the relevant code and understand the current state
2. Present a clear plan:
   - What files you would create, edit, or delete
   - What the specific changes would be
   - Any trade-offs or decisions worth calling out
3. Wait for the user to approve, adjust, or reject the plan

Only take action if the user explicitly says to go ahead.
